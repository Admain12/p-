# Bond 网卡绑定配置工具

## 简介
这是一个用于 Linux 系统的 bond 网卡绑定配置工具，使用 bash 编写，通过 nmcli 命令实现自动化网络配置。

## 功能特点
- 自动检测可用物理网卡
- 支持 7 种 bond 工作模式
- 支持 DHCP 和静态 IP 配置
- 交互式配置流程，简单易用
- 配置预览和确认机制
- 详细的执行步骤和状态显示

## 适用环境
- Linux 操作系统
- 已安装 NetworkManager (nmcli)
- 已安装 iproute2 (ip 命令)
- 需要 root 或 sudo 权限

## 支持的 Bond 模式
1. **balance-rr** (轮询模式) - 数据包依次从每个网卡发送，提供负载均衡和容错
2. **active-backup** (主备模式) - 推荐，只有一个网卡工作，其他作为备份
3. **balance-xor** (XOR 哈希分流) - 根据源/目标 MAC 或 IP 地址哈希选择网卡
4. **broadcast** (广播模式) - 所有数据包从所有网卡发送，提供最高容错
5. **802.3ad (LACP)** (动态链路聚合) - 生产推荐，需要交换机支持 LACP
6. **balance-tlb** (发送负载均衡) - 根据每个网卡的负载情况分配发送流量
7. **balance-alb** (发送+接收负载均衡) - 在 tlb 基础上增加接收负载均衡

## 运行方法

### 1. 赋予执行权限
```bash
chmod +x bond-config.sh
```

### 2. 执行脚本 (需要 root 权限)
```bash
sudo ./bond-config.sh
```

### 3. 配置流程
1. 选择 bond 接口名称 (默认: bond0)
2. 选择工作模式
3. 选择 IP 配置方式 (DHCP/静态)
4. 选择要绑定的网卡 (至少 2 块)
5. 预览配置并确认
6. 等待配置完成

## 脚本结构

### 核心功能模块
- **网卡检测**：自动识别可用物理网卡，过滤虚拟和特殊网卡
- **模式选择**：提供 7 种 bond 模式的说明和选择
- **IP 配置**：支持 DHCP 和静态 IP 配置，包括网关和 DNS
- **网卡选择**：交互式选择要绑定的网卡
- **配置执行**：停止旧连接，创建 bond 接口，添加从网卡，启动连接
- **验证**：显示 bond 状态信息

### 关键技术点
- 使用 nmcli 管理网络连接
- 使用 ip 命令获取 IP 地址信息
- 支持动态参数构建和验证
- 友好的用户界面和颜色输出
- 错误处理和边界检查

## 注意事项
1. 配置过程中网络连接会中断，请确保通过控制台或其他方式访问服务器
2. 至少需要 2 块可用物理网卡
3. 802.3ad 模式需要交换机端支持 LACP 配置
4. 建议在测试环境中先验证配置
5. 脚本会删除原有网卡连接配置

## 示例配置
```
Bond 接口名 : bond0
工作模式     : active-backup
IPv4 配置  : 静态 IP: 192.168.1.100/24
默认网关     : 192.168.1.1
DNS 服务器 : 8.8.8.8,8.8.4.4
Slave 网卡 : eth0 eth1
```
